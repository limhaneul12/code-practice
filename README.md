## 제가 느껴보았던 질문하는분들의 4유형 & 대응 전략

| 유형 | 상태 | 핵심 니즈 | 퍼실리테이터의 역할 |
|------|------|----------|---------------------|
| **① 막힌 분** | 하다가 특정 지점에서 멈춤 | 구체적 해결 | 같이 진단 |
| **② 모르는 분** | 시작점 자체가 막막 | 안심 + 첫걸음 | 길잡이 |
| **③ 불안한 분** | 만들었지만 확신 X | 검증 + 인정 | 거울 |
| **④ 확장하는 분** | 완성 후 더 깊이 | 도전 + 개념 | 동료 |

## 유형별 "물꼬 트는 첫 질문" & 실제 사례집

각 유형에게 **답을 주기 전에 먼저 던질 질문**과, 현장에서 **실제로 관찰된 패턴**을 정리했습니다. <br>
제가 대부분의 질문자의 케이스를 종합해보았을떄는 다음과 같았습니다 <br>
1. 검색 미숙 
2. 요구사항 구체화 정립 
3. 키워드 및 summarization
4. 이해 불가 

---

###  막힌 분

**첫 질문**: "어디까지는 됐고, 어디서부터 안 되나요?"
→ 스스로 문제 범위를 좁히게 함

> **이런 경우가 있더라**
> - 막혔다고 오시지만 사실 **몰라서** 또는 **해야하는 순서 정리**가 안되서 오신 분이 꽤 많으심 
> - 가장 잘 먹혔다고 생각이 드는건 할 수 있는곳까지 설명해보라고 함 단계별로 설명을 해보라고 하면서 
> - 같이 화면 보면서 설명을 하다가 → **본인이 먼저 "어 여기네요?" 하고 self 발견하는 케이스도 존재**
> - 대부분의 케이스를 볼 경우는 순서를 생각하지않고 주먹구구로 flow를 설계하다가 길을 잃은 경우가 많았음 
> - 그럴 경우 한번 자기가 생각했던걸 그려보라고 한다면 같이 어떤게 뭐였는지를 다시한번 질문 
> - 그래도 모르겠다 하면 → 레퍼런스 + "원래 목적이 뭐였죠?" 상기시키면 사고의 물꼬를 트게 하였음


### ①-A 특정 지점만 콕 집어 모르는 분 — "도구 사용법 모드"

**특징**: "다 알겠는데 **이거 어떻게 하는지만** 모르겠어요"→ flow(흐름)의 문제 X / **도구 사용법 미숙**이 대부분

**진행 흐름**
→ "어떤 점이 이해가 안 됐는지" 먼저 스스로 정리하게
→ 흐름 설계가 아니라 **기능 조작**이 막힌 것임을 확인
→ "뭘 하고 싶으세요?" → 목적 확인
→ **검색하는 방법을 같이** 찾아봄 (공식 문서, 도움말 키워드 등)
→ 답을 떠먹이기보다 → **"찾는 법"을 같이 익힘**

> **이런 경우가 있더라**
> - "Iterator를 어디서 추가해요?" 같은 **조작 질문**
> - 흐름은 머릿속에 있는데 **버튼 위치를 모름**
> - 검색 키워드만 잡아주면 → 본인이 알아서 해결
> - **"검색하는 힘"을 길러주는 게 진짜 도움**

---

### ② 모르는 분

같은 "모르겠어요"라도 결이 다름. 크게 **두 갈래**로 나뉜다.

---

#### 갈래 A — 감이 안 잡힘 (자동화 소재를 못 찾음)

**첫 질문**: "평소에 반복하는 귀찮은 일이 뭐예요?"
→ 거창한 개념 말고 본인 경험에서 출발

> **이런 경우가 있더라**
> - 개념부터 설명하면 더 막막해함 (소재 자체를 못찾아서 왜 써야하는지 몰라서)
> - 문제와 관련된 얘기를 본인 일상 업무를 꺼내면 → "아 이거 자동화 되는 거예요?" 하고 눈이 트임
> - 먼저 어떤것이 살면서 불편했거나 또는 자주 까먹는 루틴이 있는지 확인해보면 좋다 
> - 거창한 예시(KRX API)보다 **본인 메일 정리** 같은 게 와닿음 이건 예시이긴 했는데 공감대를 형성하는것이 중요 
> - 그러면서 조금씩 **사고의 물꼬**를 트는 방향으로 진행
> - 이어서 → "이걸 사용하려면 우리가 어떤 걸 진행해봐야 할까요?" 하고 **다음 행동을 스스로 떠올리게** 유도

> - 이런분들께서는 레퍼런스는 오히려 독 레퍼런스도 어느정도 알고 있는사람에게 줘야 효과 가 있다고 생각 
---

#### 갈래 B — 문제 자체를 이해 못 함 ⚠️

**첫 질문**: "이 문장을 같이 읽어볼까요? 어떻게 이해하셨어요?"
→ 도구 찾기 전에, 문제와 '동화'되는 과정이 먼저

> **이런 경우가 있더라**
> - "뭘 자동화하지? → 도구 찾자"가 아니라
>   **문제 문장 자체를 해석 못 하는** 분이 있음
> - 이때 답을 주면 X → 같이 천천히 읽으며 **키워드 짚어주기**
> - 문제 핵심을 직접 말하지 말고 읽어보면서 한문단이 완성(다 읽었을때) 그러면 이 요점은 무엇일까요? **돌려 말하며** 스스로 닿게 함

> **갈래 B 다룰 때 포인트**
> - 용어는 **직접 알려줘도 됨** (모르는 단어가 벽이니까)
> - 키워드 **같이 검색**해보기 (느려도 직접 보게)
> - 속도보다 **문맥 이해가 우선**

---

### ③ 불안한 분
**첫 질문**: "어떻게 만들었는지 설명해주실 수 있어요?"
→ 보통 자기 확신이 없어 **검증받고 싶어** 오는 경우


**진행 흐름**
→ 설명하게 하면서 "이게 원래 목적에 맞게 된 건가요?" 계속 되물음
→ 말하다 보면 **본인이 맞은 곳·틀린 곳을 스스로 찾음** (러버덕 효과)
→ 틀린 부분이 보여도 **"여기가 왜 틀렸어요"라고 직접 말하지 않음**
→ 대신 **레퍼런스 + 고민할 지점**을 던져 → 스스로 생각하게 유도

> **이런 경우가 있더라**
> - 이미 답을 아는데 **확인만 받고 싶은** 경우가 많음
> - 설명하다가 → "이 부분이 좀 찜찜한데..." 하고 **스스로 약점 짚음**
> - 테스트 한번 돌려보게 하면 → 본인이 확신 얻고 끝남

### 하지만 
> - 진짜로 틀렸는데 본인이 못 찾는 경우 ->  너무 오래 끌면 불안이 자존감 하락으로 바뀔 수 있음 
> - 불안형은 자존감이 깨지면 검증받으러 온 의미가 퇴색될 수 있음: 자기 위축이 되지 않게 하는것이.. 
> - 이럴 경우 N분 후에 힌트 강도를 한단계씩 올리는 에스컬레이션 기준이 있음 
```text 
같은 자리를 3번 이상 반복해서 보고 있음
말이 줄고 침묵이 길어짐 (사고 정지)
"아 모르겠다…" 자책성 발언 → 자존감 하락 신호
눈이 나를 자꾸 쳐다봄 (= 도움 요청 SOS) 
```

---

### ④ 확장하는 분

**첫 질문**: "지금 구조에서 뭘 더 하고 싶으세요?"
→ 이미 어느 정도 알고 온 분 → **가르치기보다 같이 고민하는 자세**

**진행 흐름**
→ 레퍼런스 + LinkedIn·관련 자료를 함께 보며 **토론**
→ 서로 **사례를 공유**하며 시야 확장
→ 내가 경험 못 한 부분은 → **솔직히 질문하며** 함께 탐구
→ "동료" 포지션 유지 (선생 X)

> **이런 경우가 있더라**
> - 가르치려 들면 오히려 흥미 떨어짐
> - "어디에 넣으면 좋을까요?" 하고 **같이 고민하면** 신나서 더 파고듦
> - 동료처럼 대하는 게 핵심 (AI 연동, 멀티 분기 등으로 자연스럽게 확장)

## 나만의 질문 템플릿 

이걸 현장에서 빠르게 쓰려면, **첫 1분 진단 질문** 만들어보았음

> **"안녕하세요! 혹시 어디에 가까우세요?"**
> ① 하다가 막혔어요
> ② 어디서 시작할지 모르겠어요
> ③ 만들긴 했는데 확인받고 싶어요
> ④ 더 해보고 싶은 게 있어요

→ 한 문장으로 유형 분류 후 모드 전환

## 유형별 질문 템플릿

### ① 막힌 분 — "범위 좁히기 모드"
- "어디까지는 됐고, 어디서부터 안 되나요?"
- "화면 한번 같이 볼 수 있을까요?"
- "원래 하려던 목적이 뭐였죠?"
- "절차별로 설명해주실 수 있으실까요?

### ② 모르는 분 — "출발점 찾기 모드"
**갈래 A (감 X)**
- "평소에 반복하는 귀찮은 일이 뭐예요?"
- "그거 손으로 하면 몇 분 걸려요?"

**갈래 B (문제 이해 X)** ⚠️
- "이 문장 같이 읽어볼까요? 어떻게 이해하셨어요?"
- "이 키워드는 어떻게 생각하시나요?"
- "선생님 께서는 평소에 귀찮았던것 또는 자주하시던 루틴이 있을까요"

### ③ 불안한 분 — "self 검증 모드"
- "어떻게 만들었는지 설명해주실 수 있어요?"
- "이게 원래 목적에 맞게 된 걸까요?"
- "한번 테스트로 돌려볼까요?"
- "어떤 의도로 만들었는지 목적와 당위가 궁금해요"

### ④ 확장하는 분 — "동료 토론 모드"
- "지금 구조에서 뭘 더 하고 싶으세요?"
- "이런 사례 봤는데, 어떻게 생각하세요?" (레퍼런스 공유)
- "저도 이 부분은 안 해봤는데, 같이 파볼까요?"

## 질문 설계의 숨은 패턴

| 유형 | 질문의 성격 | 누가 답을 찾나 |
|---|---|---|
| ① 막힌 분 | 범위 묻기 | 본인이 좁힘 |
| ②-A | 경험 묻기 | 본인 일상에서 |
| ②-B | 이해 확인 | 같이 읽으며 |
| ③ 불안한 분 | 설명 시키기 | 말하다 self 발견 |
| ④ 확장하는 분 | 의견 묻기 | 함께 토론 |

→ ①②③은 **"질문을 던져서 본인이 답하게"**
→ ④만 **"내 의견도 같이 꺼내서 토론"** (동료니까)

검색 키워드 잡는 법을 알려주면 스스로 해결하는 힘이 생길 수 있게. ("물고기 주지 말고 낚시법 알려주기"!!)
